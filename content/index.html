Architecture

How it works under the hood

Every design choice was made to minimize latency and maximize throughput without relying on third-party libraries.

SO_REUSEPORT Multi-Threading

Each worker thread creates its own listening socket bound to the same port. The Linux kernel distributes incoming connections across all threads with zero userspace locking on the hot path.

Zero-Copy sendfile()

Static files are transferred directly from disk to socket via the sendfile() syscall, bypassing userspace memory entirely.

Request Validation

Enforces RFC 9112 Host header requirements and blocks path traversal attacks before any file system access.

Keep-Alive + TCP_NODELAY

Persistent HTTP connections with Nagle's algorithm disabled for instant small-response delivery.

Custom Router

Register lambda-based API routes that take priority over static file serving. Simple map-based dispatch for O(log n) lookup.

Supported Methods

Full CRUD over HTTP

The server implements five standard HTTP methods, each with proper status codes and error handling.

GET

Serve Static Files

Returns the requested file from the root directory with correct MIME types. Uses sendfile() for zero-copy transfer.

curl http://localhost:3490/index.html
POST

Create a New File

Writes the request body to a new file at the given path. Returns 409 if the file already exists.

curl -X POST -d "content" http://localhost:3490/new.txt
PATCH

Append to a File

Appends the request body to an existing file. Returns 404 if the file does not exist.

curl -X PATCH -d " more" http://localhost:3490/new.txt
DELETE

Remove a File

Deletes the file at the specified path. Only regular files can be deleted, not directories.

curl -X DELETE http://localhost:3490/new.txt
HEAD

Inspect Headers

Returns identical headers to a GET request but with no response body. Useful for checking file existence.

curl -I http://localhost:3490/index.html
